# Страница трека / Player

В задании есть Player Implementation (global player)

Этот компорнент коллеги реализовали уже давно. Музыка играет, при переходах по страницам не выключается, переключение треков работает. В общем красота.

По заданию плеер должен уметь

#### _Queue (playback queue)_

_The user can open the playback queue list._
_Tracks are added to the queue when pressing Play or "Play all"._
_Previous/Next switch between tracks in the queue._

Кроме этого, инженер, который проектировал компонент плеера следовал общепрнятой логике — в мобильной версии убрать лишние контролы, оставить только важное.

Т.е. задача — показать очередь и дать доп контролы на мобилке.

Обсудили с командой в чате 2 возможных пути:

1. Самому плееру сделать extend. Иконка, которая открывае его на весь экран, и там очередь и контролы на мобилке.

2. Сделать по аналогии с существующими музыкальными треками страницу для каждого трека:
   — обложка,
   — название,альбом и артист
   — визуализация,
   — список треков
   — в мобильной версии контролы плеера, которые не влезли в панельку плеера

У страницы роут вида track/:id

Решил пойтип по второму пути, кажется архитектурно правильнее сделать страничку трека по аналогии с музыкантами и альбомами.

### Реалиация

На странице из адреса достаем ID, с помощью httpResource достаю данные о треке.
Выводим всю доступную инфу, для даты используем встроеный pipe.
