убрал дублирование запроса.

с разделением данных, прилетающих с бека и отфильтрованых-отсортированных для отображения.
Сделал проверки, чтобы понимать что больше песен ждать не стоит.

Разделение результатов и отображения помогает в части случаев сортировку и фильтрацию делать без запросов к АПИ.

Сортирвка по имени и дате, а так же фильтрация по длительности — это операции с массивом.

Фильтрация по жанру и популярности — доп запросы к апи.

Добавляем в сервис работу с параметром tags, куда передается жанр. Добвляем из primeNG множественную выбиралку. делаем сигнал для выбранных жанров.

С сортировкой хотелось поступить также – использовать параметр orde. Но наш бек падает с ошибкой 400, если добавить такой параметр в searchService. UI для пользователя бесшовный — он выбрает просто порядок сортировки. В объекте с параметрами я добавляю свойство, по коророму можно понять, требуется ли эта сортировка запроса на бек. Добавляю методы, которые работают в случае, если запрос на бек нужен. Делаю метод который подкидывает нужные парметры в сущесвующий поисковый запрос — такой формат наш бек отрабатывает корреткно.

Для фильтрации по жанрам надо получать список жанров по запросу, у нас была такая штука сделана в home service. Раз уж требуется и в поиске, есть смысл сделать отдельный сервис для работы с жанрами. На самом деле это реваренс в сторону структуры кода. потому что на беке есть эндпоит для работы с жанрами, но возвразает он не такой список, как ожидает jamendo. Поэтому для корреткной работы ьесть сервис который возвращает observable чтобы с этим работать как положено или толко в реализации сервиса видно чо это мок и его легко будет заменить на работу с запросами.

В задании сказано, что поиск должен реагировать на изменение каждой буквы.
Почитал доку по jamendo. Там есть энпоинт, который отвечает за автодополнение. Базовая идея была сделать в поиске при вводе работу с автодополнением, а по сабмиту открывать станицу поиска. В нашей реализации бека этой фишки нет, поэтому прихожу к live-поиску, как это описано в задании.

Изучаю вопрос, останавливаюсь на идее, проверять какая страница сейчас открыта, если это не поиск — то нужно к поиску перейти, когда пользователь закончил печатать, если поиск — то менять запрос. Осталное уже в сделано, по идее много менять не придется.
